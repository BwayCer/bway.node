_parseUrlRelative(idInfo, prefix, path) → {Array}

解析網址相對關係： 協助 _infoForm#_parseId 、 _infoForm#_parseDepId 取得大部分的識別碼資訊。

注意，可能有網址陷阱！
1.
網址 `/ABC` 可通過解析； 但若網址 `http://ABC` 則不給過，必須為 `http://ABC/` 才可以。
試想 `/` 網址可是為 `http://example.com/` 的最後一撇斜線呢！
2.
若網址為 `/ABC/EFG` ，則相對路徑 `../Hi` 為 `/Hi`；
若網址為 `/ABC/EFG/` ，則相對路徑 `../Hi` 為 `/ABC/Hi` 。
Parameters:
Name Type Description
idInfo Array 識別碼資訊。
prefix String 前綴符。 用已得知的資訊減少正規的判別網址相對絕對路徑關係。
path String 待解析路徑。
